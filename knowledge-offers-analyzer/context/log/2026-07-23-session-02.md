## Design decision (why the false-positive handling looks like this)

Absence from one profile's search ‚Č† removal. The systematic corruptors and their handling:
`submittedWithin` profiles are **never** detection-eligible (window aging would produce a wall
of dom‚Čąwindow false events); truncated result pages (total > ids, or unknown total at the
100-cap) make a profile-cycle ineligible; a candidate must still be **covered** by an eligible
profile's filters (pure `profileCovers`); 24h **grace** + a resurrection path voids transient
hiding. Residual noise (price-*increase* band exits, long-sitters) biases `k` **upward** toward
1.0 ‚ÄĒ i.e. under-correction ‚ÄĒ the right failure direction for a correction whose falsification
criterion is `k ‚Č• 0.97`. `disappeared_at` = last confirmed sighting (not detection time) so
grace never skews DOM. Backfill-wave events are quarantined via `detection_mode='backfill'`.

## US4.1b ‚ÄĒ daily market sweep (added same session, operator decision)

T012 resolution: the operator rejected model-pinned niche profiles ‚ÄĒ their real niche is
market-wide ("–≤—Ā—Ź –ö–ł—ó–≤—Č–ł–Ĺ–į ‚Č§ $15k, 2010+"), measured live at **~17,898 active matches** (one
search, reading the new free `total`). No plain persistent profile can be detection-eligible at
that scale (100-id page cap, no paging in the poll), so SPEC-004 was extended with **US4.1b**:

- `ProfileFilters.sweep?: boolean` (jsonb ‚ÄĒ no migration); sweep profiles are **excluded from
  the 10-min poll** and never ingest/evaluate.
- `SweepService` (polling module, `@Cron '30 3 * * *'`): pages `source.search({...toQuery, page})`
  per sweep profile until empty/short page or ids ‚Č• total (`MAX_SWEEP_PAGES=250` safety cap ‚Üí
  incomplete); each page budget-gated; `RateBudgetExhaustedError` aborts the whole run (shared
  budget); **detection only on a complete crawl** ‚ÄĒ `processCycle(ids, [profile],
  SWEEP_GRACE_HOURS=30)` + passive `'disappeared'` outcomes. 30h grace ‚áí one missed/failed
  sweep (or intra-sweep pagination drift) never records an event; two consecutive misses do.
- `processCycle` gained the optional `graceHours` param (default `GRACE_HOURS=24`).
- Config: profile "Sweep ‚ÄĒ –ö–ł—ó–≤—Č–ł–Ĺ–į, –ī–ĺ $15000, 2010+" (`sweep: true`, `dealerPolicy: 'label'`
  so dealer listings stay covered), enabled.

  zero-cost framing for this profile class only; regular-profile detection stays zero-request.
