## Design decision (why the false-positive handling looks like this)

Absence from one profile's search ≠ removal. The systematic corruptors and their handling:
`submittedWithin` profiles are **never** detection-eligible (window aging would produce a wall
of dom≈window false events); truncated result pages (total > ids, or unknown total at the
100-cap) make a profile-cycle ineligible; a candidate must still be **covered** by an eligible
profile's filters (pure `profileCovers`); 24h **grace** + a resurrection path voids transient
hiding. Residual noise (price-*increase* band exits, long-sitters) biases `k` **upward** toward
1.0 — i.e. under-correction — the right failure direction for a correction whose falsification
criterion is `k ≥ 0.97`. `disappeared_at` = last confirmed sighting (not detection time) so
grace never skews DOM. Backfill-wave events are quarantined via `detection_mode='backfill'`.

## US4.1b — daily market sweep (added same session, operator decision)

T012 resolution: the operator rejected model-pinned niche profiles — their real niche is
market-wide ("вся Київщина ≤ $15k, 2010+"), measured live at **~17,898 active matches** (one
search, reading the new free `total`). No plain persistent profile can be detection-eligible at
that scale (100-id page cap, no paging in the poll), so SPEC-004 was extended with **US4.1b**:

- `ProfileFilters.sweep?: boolean` (jsonb — no migration); sweep profiles are **excluded from
  the 10-min poll** and never ingest/evaluate.
- `SweepService` (polling module, `@Cron '30 3 * * *'`): pages `source.search({...toQuery, page})`
  per sweep profile until empty/short page or ids ≥ total (`MAX_SWEEP_PAGES=250` safety cap →
  incomplete); each page budget-gated; `RateBudgetExhaustedError` aborts the whole run (shared
  budget); **detection only on a complete crawl** — `processCycle(ids, [profile],
  SWEEP_GRACE_HOURS=30)` + passive `'disappeared'` outcomes. 30h grace ⇒ one missed/failed
  sweep (or intra-sweep pagination drift) never records an event; two consecutive misses do.
- `processCycle` gained the optional `graceHours` param (default `GRACE_HOURS=24`).
- Config: profile "Sweep — Київщина, до $15000, 2010+" (`sweep: true`, `dealerPolicy: 'label'`
  so dealer listings stay covered), enabled.
- Cost: ~179 pages/day ≈ **5,400 req/mo** (ADR-0009 pool) — a deliberate amendment to the
  zero-cost framing for this profile class only; regular-profile detection stays zero-request.
- Implementation: SDD updates + flag/exclusion/grace param (orchestrator), `SweepService` +
  7 tests delegated → Sonnet. Final: tsc clean, jest **27 suites / 176 tests** green.
