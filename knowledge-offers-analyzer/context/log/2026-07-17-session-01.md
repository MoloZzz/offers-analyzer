## Done this step — M2 (analytic mileage correction, percentage model)

Operator chose the **simpler** percentage model. `resolveBenchmark` now returns `mileageAware`
(true when the matched cohort was mileage-banded). New `src/modules/valuation/mileage.ts`:
pure fns (`expectedMileageK`, `mileageAdjustmentPct`, `adjustFairForMileage`) + `MileageAdjuster`
service. When the benchmark is **not** mileage-aware, fair value shifts by
`(expected − actual)/10 × per10kPct` %, capped ±`maxAdjPct`, `expected = age × annualK`. Config
(`configuration.ts` + `.env.example`): `MILEAGE_ANNUAL_K=15`, `MILEAGE_PER_10K_PCT=2`,
`MILEAGE_MAX_ADJ_PCT=20`. Wired into `poll.service` + `query.service` (both inject `MileageAdjuster`,
provided/exported by `ValuationModule`). Pure fns unit-tested (`test/unit/mileage.spec.ts`).

**Verify:** `tsc` clean; `jest` 18/18 pass (4 suites). No schema change.

Sandbox note (again): the bash mount truncated *every* file-tool-edited source (configuration,
valuation.module, poll.service, query.service, cohort) — re-flushed each via bash `cat >` with the
store content, then tsc/jest passed. Confirmed via file-tool Read that the store (→ user's disk) holds
the correct content.
