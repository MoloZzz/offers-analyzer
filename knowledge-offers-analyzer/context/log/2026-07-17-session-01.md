## Spec 002 authored — auto-calibration & learning

Operator asked whether calibration/learning was "ready". Honest answer: no — R1 only *recommends* a
threshold (manual apply) and the scorer is fixed rule-based constants; crucially **we record no
outcomes**, so there's nothing to learn from. Wrote a full SDD spec for it:
`specs/002-auto-calibration-learning/` (spec.md + plan.md + tasks.md).

Shape: (E1) make scoring read a **versioned active `ParameterSet`** (seeded from today's config →
identical behavior, live-tunable, rollbackable); (E2/US1, the MVP) **capture outcomes** — 👍/👎 buttons +
`/outcome` + passive signals (disappeared/price-drop/time-on-market), report shows realized precision;
(E3/US2) **auto-calibrate `minDealScore`** per profile toward a volume-corridor / precision target,
bounded + versioned + `/revert`, propose-only by default; (E4/US3) **learn weights** (penalties,
mileage, condition) propose-only with evidence. Guardrails baked in: transparent (no black-box ML),
bounded/reversible, freeze on thin data, **zero API budget** (stored-data only), safe cold start.

Key honesty point captured in the spec: **selection bias** — we only observe outcomes for *alerted*
listings, so learning is scoped to precision on that set (near-misses/passive as a recall proxy), not a
full-label model. Needs **ADR-0005** at implementation time + a supersession sweep (weights are
currently documented as fixed constants). Linked from `specs/README`, backlog (🟣 epic), this log.

## Pre-implementation fixes (before spec 002)

- **Null `lastScore`/`lastEvaluatedAt` bug.** Root cause: a listing is `recordSeen` before it's scored;
  if evaluation is interrupted (budget exhausted mid-cycle, or a source error in `resolveBenchmark`),
  it's saved unscored, and next cycles treat it as "known" → the re-observe path only scored on a price
  drop → null stuck forever. Fix in `poll.service`: `reobserve` now also evaluates when
  `existing.lastScore == null` (not just on a drop), and phase-1 `stale` selection puts never-scored
  listings first. (Dealer-`exclude` is a separate, intended non-scoring case; operator uses `label`.)
- **Threshold raised.** `minDealScore` 0.15 → **0.63** (profiles JSON + example + `configuration.ts`
  default + `.env.example`). Rationale: score = discount%/30 at full confidence, so 0.63 ≈ **19%** below
  market minimum; risk flags / thin data require a bit more discount (intended). tsc clean; jest 28/28.
- Confirmed to operator: mileage **is** in scoring (M1 banded cohort + M2 correction); description **is**
  in scoring (condition → red-flags) — both in the alert path, not just `/check`.
